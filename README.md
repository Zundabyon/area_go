**AreaGo** は、エリアトラウト（管理釣り場）専用の**釣果管理 PWA アプリ**です。
施設の検索・登録から、ルアーのコレクション管理、釣果の記録まで一元化します。

## サービス概要

AreaGo は、**どの施設で・どのルアーで・どんな魚が釣れたか**を記録し、振り返ることができるサービスです。

釣果を単なる記録として残すのではなく、**ルアーのカラーや浮力、釣れた水深まで含めた"釣りの再現性"を高めること**を目指しています。

---

## このサービスを作った理由

エリアトラウトを楽しむ中で、「あの日あの施設で釣れたルアーは何だったか？」「水深はどのくらいだったか？」という情報が散らばり、次の釣りに活かせないもどかしさを感じていました。

既存のスコア記録アプリや釣果アプリの多くは、**魚種やサイズの記録が中心**で、エリアトラウト特有の「ルアーの種類・カラー・浮力・使用レンジ」まで細かく残せるものが見当たりませんでした。

**釣果を再現性ある形で記録・活用できるアプリ**を自分で作りたいと考え、開発に至りました。

## 使い方

### 1. アカウント作成

トップページから新規登録を行います。メールアドレスとユーザー名を入力してアカウントを作成します。

### 2. 施設を登録

「施設検索」から管理釣り場を検索し、登録します。

- Google Maps Places API によるキーワード検索
- 都道府県・エリアからの絞り込み
- 施設詳細（住所・電話番号・公式サイト）

### 3. ルアーを登録

「ルアー管理」から使用ルアーをコレクションに登録します。

- タイプ別登録（スプーン / クランクベイト / ミノー / その他）
- 表カラー・裏カラーをカラーピッカーで設定
- 浮力（ハイフロート 〜 シンキング）をスライダーで設定
- ウェイト（g）記録

### 4. 釣果を記録

「釣果記録」から、釣れた内容を詳細に残します。

- 魚種・サイズ・使用ルアー
- 釣れた日時・放流時間
- 地図ピンで釣れた地点を記録
- 水深（m）記録
- Canvas で水断面図とルアー軌跡を手書き保存
- メモ欄

### 5. 釣果を振り返る

ダッシュボードから、施設・ルアー・魚種ごとに釣果を振り返ることができます。

## ターゲット層

### メインターゲット

- 20〜40代のエリアトラウト（管理釣り場）愛好者
- ルアーを複数所持しており、釣果とルアーの相関を把握したい人
- 施設ごとの攻略パターンを記録・蓄積したい人
- スマホで手軽に釣り場から記録を残したい人

### この層をターゲットにした理由

エリアトラウトは、フィールドコンディションや使用ルアーの微差が釣果に直結するため、**記録の精度が釣りの上達に直接影響する**ジャンルです。また、スマートフォン普及率が高い世代であり、釣り場でのモバイル入力にも抵抗が少ないと考えました。

## 差別化ポイント

|              | 一般的な釣果アプリ       | AreaGo                                 |
| ------------ | ------------------------ | -------------------------------------- |
| 主な目的     | 釣果サイズ・魚種の記録   | ルアーと釣果の関係を再現性ある形で記録 |
| ルアー管理   | タイトル名のみ           | カラー・浮力・ウェイトまで管理         |
| 対応ジャンル | 汎用（海釣り・渓流など） | エリアトラウト専用                     |
| 入力形式     | テキスト中心             | Canvas による水断面図・地図ピン        |

既存の釣果アプリが **「何が釣れたか」** の記録に重点を置いているのに対し、AreaGo は **「なぜ釣れたか・どうすれば再現できるか」** の記録に特化している点が最大の差別化ポイントです。

## 技術的に工夫した点

### Google Maps Places API × 施設登録フロー

管理釣り場は数が多く、施設名の揺れ（正式名・通称）もあるため、ユーザーが手動入力すると登録精度が下がるという課題がありました。

Google Maps Places API を活用し、キーワード検索 → 候補一覧から選択 → 住所・緯度経度を自動セットするフローを実装することで、入力ステップを大幅に削減しました。

### Canvas を使った水断面図の記録

釣果の「水深」と「レンジ」は数値だけでは伝わりにくいため、Canvas を用いた水断面図の手書き入力機能を実装しました。描画データは JSONB 形式でデータベースに保存し、再表示・編集が可能です。

### ルアーカラーの hex 値管理

釣りにおいてルアーのカラーは重要な変数です。テキスト表記では曖昧になるため、カラーピッカーで選択した色を hex 値でそのまま保存し、一覧画面では実際の色をビジュアルで確認できる UI を実装しました。

## 技術選定理由

### フロントエンド：Hotwire（Turbo + Stimulus）

Rails 8 のデフォルトスタックであり、SPA ライクな操作感をサーバーサイドレンダリングで実現できます。地図操作やカラーピッカーなど、インタラクションが必要な箇所は Stimulus コントローラーで対応し、JavaScript の記述量を最小限に抑えました。

### CSS：Tailwind CSS v4

Rails との相性が良く、カスタムアニメーションやグラデーション表現も柔軟に組み込めます。水・自然をモチーフにしたカラーパレットを `@theme` で一元管理し、デザインの統一性を保ちやすい点も採用理由です。

### バックエンド：Ruby / Ruby on Rails

施設・ルアー・釣果のリレーションが明確な CRUD 中心の設計のため、Rails の MVC パターンと Active Record との相性が高く、開発効率・保守性の観点から採用しました。

### データベース：PostgreSQL（Neon）

- **Render との親和性**：Render での PostgreSQL 運用実績が豊富でデプロイ・運用が容易
- **Neon のサーバーレスアーキテクチャ**：無料枠で本番運用が可能。初期コストを抑えられる
- **JSONB 型のサポート**：Canvas 描画データを `fishing_method_data` カラムに JSONB で保存するユースケースに最適

### 地図：Google Maps JavaScript API / Places API

管理釣り場の検索・登録・釣れた地点のピン記録のために採用しました。Places API による施設検索とオートコンプリートにより、正確な施設情報（名称・住所・緯度経度）を自動取得できます。

### インフラ：Docker + Render

`Dockerfile.dev`（開発環境）と `Dockerfile`（本番環境）を分離し、ローカル・本番間の環境差異を最小化しました。Render は GitHub 連携による自動デプロイが容易で、無料枠で本番運用が可能です。

### 認証：Devise

Rails のデファクトスタンダードであり、メール認証・パスワードリセットなどの認証フローが標準実装済みです。セキュリティの担保と開発速度の両立を実現しています。

### 主なテーブル設計の意図

- `users` ← `lures`：1ユーザーが複数のルアーを所持できる1対多の関係
- `users` ← `catch_records`：1ユーザーが複数の釣果を記録できる1対多の関係
- `catch_records` に `lure_id`・`facility_id` を持たせ、ルアー × 施設 × 釣果の三者を紐付け
- `fishing_method_data` を JSONB で持つことで、Canvas 描画データを柔軟に保存

## 開発背景

このアプリは、私が**エリアトラウトというニッチな釣りジャンルの課題**を解決しようと開発したものです。

エリアトラウトは、ルアーの選択・カラー・レンジ管理など、細かいデータの積み重ねが釣果に直結するジャンルです。しかし「記録」に特化したアプリが存在しないため、多くのアングラーがメモ帳や記憶に頼っているのが現状です。

RUNTEQ というプログラミングスクールで Ruby と Rails を学ぶ中で、**自分が実際に困っていることをアプリで解決する**という経験を積みたいと考え、開発に至りました。

このアプリが、釣りを楽しむすべての人の

> 「また行きたい」
> 「次はこのルアーで試したい」
> 「あの日と同じ条件で釣れた！」

そんな体験のきっかけになれば、とても嬉しいです。

まだまだ発展途上のアプリではありますが、愛情を込めて作っています。ぜひ触っていただき、フィードバックをいただけると嬉しいです。
